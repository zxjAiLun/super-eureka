## 1. Commands run (Commit 5 gate)

```bash
cargo run --release -- bench smoke                                      # default profile = reference
cargo run --release -- bench standard --mode disabled --profile reference
cargo run --release -- bench standard --mode disabled --profile current
cargo run --release -- bench throughput --mode disabled --profile reference --nodes 100000 --repeat 3
cargo run --release -- bench throughput --mode disabled --profile current    --nodes 100000 --repeat 3
cargo run --release -- perft 5                                        # must be 4,865,609
```

All runs use `--mode disabled` (TT disabled) so the node counts isolate the
killer/history ordering effect and are not dominated by TT reuse (warm-TT
results are treated as *recorded-only* per spec §7). The fixed-node
throughput pair is run **separately** for `reference` and `current` and
compared fixture-by-fixture on **median NPS**.

bestmove, and a stepwise-legal PV — i.e. the §6 hard-correctness items
hold. (The fixed-depth completeness + restoration invariants are enforced by
`bench.rs` `validate` for every mode.)

## 5. Single-fixture > 20% regression check (§7)

**None.**

- **Node regressions:** every fixture's `current` node count is *lower* than
  `reference` (all reductions positive). No fixture searched *more* nodes.
- **NPS regressions:** all per-fixture NPS drops are ≤ 7.5% (≤ 15%
  tolerance). No fixture exceeds 20%.

The two smallest node reductions (kqk 0.2%, krk 1.1%) are *improvements*,
just small — those are low-branching king+queen/rook endgames with
near-forced lines where killer/history ordering has little to reorder. They are
expected and are **not** regressions.

## 9. What Commit 5 changed (scope)

- `src/engine/bench.rs`: added an **optional** `--profile reference|current`
  flag (default `reference`, so every existing M4.0 command reproduces
  exactly). `reference` drives the original M4.0 entry; `current` drives the
  profile-aware entry with `SearchProfile::Current`. `bench_result` and
  `bench_summary` now emit `profile=`. No search semantics changed.
- `README.md`: added a short `--profile` usage note.
- `src/engine/search.rs` and `src/uci.rs`: **not modified** (no production
  search-semantics change). The retained M4.0 entry
  (`search_best_move_with_history_and_tt`) stays as the `reference` path and
  is still exercised by the M4.0 reference tests.
- `docs/benchmarks/m4.1-quiet-move-ordering.md`: this report.

> Note: warm-TT mode remains available but is **recorded-only** per spec §7
> (TT reuse dominates there and is not the primary proof of the killer/history
> effect). The required Commit-5 gate commands all use `--mode disabled`.
